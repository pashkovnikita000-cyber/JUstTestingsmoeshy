  <action>
    Create Dockerfile at project root.

    Base image: python:3.12-slim (per CLAUDE.md stack).

    Layer order (cache-friendly, fat layers first):
    1. FROM python:3.12-slim, WORKDIR /app
    2. COPY requirements.txt . then RUN pip install --no-cache-dir -r requirements.txt — this layer caches as long as requirements.txt doesn't change
    3. COPY bot/ ./bot/ — source code, changes often
    4. RUN mkdir -p /app/data — SQLite volume mount point; init_db() also creates it but mkdir here ensures permissions
    5. Create non-root user: RUN useradd -m -u 1000 botuser AND chown -R botuser:botuser /app, then USER botuser
    6. CMD ["python", "-m", "bot.main"]

    No EXPOSE needed (long-polling, no inbound ports).
    No ENV in Dockerfile — all env vars come from .env or Railway Variables at runtime.
    No COPY .env (that file is gitignored and must not enter image).

    Create docker-compose.yml at project root for local development only:
    - version: "3.9" (or omit version key for compose v2 compatibility)
    - services.bot: build: . (context dot), env_file: .env, volumes: - ./data:/app/data, restart: unless-stopped

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Container → host filesystem | Volume mount ./data:/app/data — DB file accessible on host; acceptable for personal bot |
| Image → runtime secrets | .env / Railway Variables not baked into image — correct pattern |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-10 | Information Disclosure | Secrets baked into Docker image | mitigate | No COPY .env in Dockerfile; no ENV BOT_TOKEN in Dockerfile; all secrets injected at runtime via env_file or Railway Variables |
| T-02-11 | Elevation of Privilege | Container running as root | mitigate | Dockerfile creates botuser (UID 1000) and switches USER botuser before CMD |
| T-02-12 | Denial of Service | DB lost on Railway restart | mitigate | Railway Persistent Volume mounted at /app/data (user setup step); documented in README as critical step |
| T-02-SC | Tampering | Base image supply chain | accept | python:3.12-slim is official Docker Hub Python image; acceptable risk for personal bot |
</threat_model>
