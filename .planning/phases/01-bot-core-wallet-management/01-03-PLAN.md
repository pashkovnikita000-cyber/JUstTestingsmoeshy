<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Telegram user → bot handlers | Аргументы команд и сообщения диалога — недоверенный ввод |
| user_id → данные | Каждый пользователь должен видеть только свои кошельки |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-08 | Spoofing | Все wallet-команды | mitigate | Каждый хендлер под `@restricted` (whitelist из 01-01) |
| T-01-09 | Tampering / Information Disclosure | Доступ к чужим кошелькам | mitigate | Все запросы фильтруют по `user_id` из update; get/remove не возвращают/не трогают чужое (WALLET-05) |
| T-01-10 | Tampering | Невалидный адрес в /addwallet | mitigate | `validate_address` в состоянии ASK_ADDRESS до сохранения (WALLET-02) |
| T-01-11 | Tampering | SQL-инъекция через адрес/имя | mitigate | Только параметризованные запросы aiosqlite (?-плейсхолдеры), без f-string в SQL |
| T-01-12 | Denial of Service | Много кошельков → шквал запросов к Etherscan в /wallets | mitigate | Последовательная загрузка с троттлом 0.25 сек (D-06), одна цена на список |
| T-01-13 | Denial of Service | Сбой Etherscan роняет хендлер | mitigate | try/except вокруг сетевых вызовов; деградация (баланс «недоступен»), кошелёк всё равно сохраняется/показывается |
</threat_model>
