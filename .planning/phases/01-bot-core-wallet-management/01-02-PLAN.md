<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Etherscan API → bot | Внешние данные (баланс/цена) — недоверенный ввод |
| User-provided address → Etherscan | Адрес от пользователя уходит в запрос |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-04 | Tampering | Адрес пользователя в URL запроса | mitigate | `validate_address` (строгая regex 0x+40hex) до любого запроса; невалидный адрес не уходит в Etherscan (WALLET-02) |
| T-01-05 | Denial of Service | Превышение лимита Etherscan (5 req/sec) → бан ключа | mitigate | Троттл >=0.25 сек между запросами (D-06) |
| T-01-06 | Tampering | Некорректный/неполный ответ API (status!="1", пустой result) | mitigate | Парсеры проверяют наличие полей и кидают понятную ошибку вместо краша при отсутствии result |
| T-01-07 | Information Disclosure | ETHERSCAN_API_KEY в логах/URL | accept | Free-tier ключ, низкая ценность; не логировать полный URL |
</threat_model>
